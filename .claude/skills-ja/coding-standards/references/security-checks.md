# セキュリティチェックパターン

Last reviewed: 2026-03-21

## Stable Patterns

偽陽性率が低く、grepまたは静的解析で検出可能なパターン。

### ハードコードされたシークレット
- ソースコード内で文字列リテラルとして代入された認証情報、APIキー、トークン
- パスワードが埋め込まれた接続文字列
- ソースファイルに保存された秘密鍵や証明書
- 検出方法: 代入演算子付近の高エントロピー文字列、一般的なキー名（`password`、`secret`、`api_key`、`token`、`private_key`）、プラットフォーム固有のトークン形式を検索

### SQL文字列結合
- 文字列結合や変数を含むテンプレートリテラルで構築されたSQL文
- 検出方法: SQLキーワード（`SELECT`、`INSERT`、`UPDATE`、`DELETE`）と文字列結合演算子または変数参照を含むテンプレートリテラルの組み合わせを検索

### 動的コード実行
- 動的な入力を引数とする `eval()`、`Function()`（JS）、`exec()`、`compile()`（Python）等の使用
- 変数パスによる動的importやrequire
- 検出方法: 引数が静的リテラルでないこれらの関数呼び出しを検索

### 安全でないデシリアライゼーション
- 信頼できない入力での`pickle.loads()`、SafeLoaderなしの`yaml.load()`、`marshal.loads()`
- `JSON.parse()`後に`eval()`や`Function()`で直接使用
- 検出方法: 安全なローダー設定なしで外部入力を受け入れるデシリアライゼーション呼び出しを検索

### パストラバーサル
- ユーザー提供入力からサニタイズなしで構築されたファイルシステムパス
- リクエストパラメータがファイル読み書き操作に流れるパターン
- 検出方法: パス引数にリクエストパラメータ、クエリ文字列、ユーザー入力変数を含むファイル操作を検索

### CORSワイルドカード
- 本番設定で`Access-Control-Allow-Origin`が`*`に設定
- ワイルドカードoriginで設定されたCORSミドルウェア
- 検出方法: ワイルドカード値のCORS設定を検索

### 非TLS URL
- ソースコード内に埋め込まれた本番エンドポイント向けHTTP（非TLS）URL（設定ファイル、テスト、ドキュメントを除く）
- 検出方法: ソースファイル内の`http://`パターンを検索（localhost、設定ファイル、テスト、ドキュメントを除外）

## Trend-Sensitive Patterns

Updated: 2026-03-21
Sources: OWASP Top 10:2025, DryRun Agentic Coding Security Report (2026-03)

### AI生成コードにおけるAccess Controlの欠落
- 認証ミドルウェアなしで定義されたエンドポイントやルートハンドラ
- 認可検証なしのリソースアクセス操作（読み取り、更新、削除）
- 昇格された権限なしでアクセス可能な管理操作や破壊的操作
- 最近の研究でこのパターンがAI生成コードで高い発生率を示す — 高優先度のレビュー対象として扱う

### 例外条件の不適切な処理（OWASP A10:2025）
- レスポンスに内部システム詳細（スタックトレース、データベースエラー、ファイルパス）を露出するエラーハンドラ
- フェイルオープン（エラー時にアクセスを許可または検証をスキップ）するエラーハンドラ
- セキュリティ上重要な操作（認証、認可、暗号操作）でのエラーハンドリング欠如

### ソフトウェアサプライチェーンパターン（OWASP A03:2025）
- バージョン固定なしでインポートされた依存関係
- セキュリティ上重要な機能に対する非推奨または保守されていないパッケージの使用
- 検出方法: 依存関係マニフェストのバージョン未固定と既知の非推奨パッケージを確認
